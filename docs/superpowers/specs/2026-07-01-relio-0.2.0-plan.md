# 0.2.0 — Agentic + Data + Accounts

**Date:** 2026-07-01
**Status:** Approved (build 1 → 2 → 3)

## W1 — Autonomous tool-calling loop (headline)
The LLM decides which exposure-map tools to call, we execute them, feed results
back, and loop until a final answer.
- `LLMProvider.complete_with_tools(messages, system, tools) -> {"tool_calls":[...]}
  | {"text": ...}` (optional capability; default `NotImplementedError`).
- `FakeProvider` implements it deterministically (offline tests).
- `Agent.run(task, max_steps)` runs the loop over the agent's **tool slice**;
  **destructive tools are never auto-executed** (blocked, needs `confirm`).
- Claude/OpenAI implementations (lazy, untested here).

## W2 — Data-app query
- `where` gains operators: `field__gt/gte/lt/lte/contains` (plus exact); add
  `order_by` (`-field` = desc), `limit`, `offset` to `query`.
- Graph `neighbors`/`traverse` accept a `scope` and filter by it; the graph route
  enforces the principal.

## W3 — Accounts / auth polish
- **Refresh tokens** — `issue_tokens` → `{access, refresh}`; `POST /auth/refresh`.
- **Password reset** — `POST /auth/reset-request` (returns a reset token) +
  `POST /auth/reset`.
- **GitHub OAuth** — `GitHubOAuth` (same shape as `GoogleOAuth`).
- **Rate-limited login** — optional limiter on the accounts router.

## Out of scope (defer)
Email *sending* (reset/verify return tokens; delivery is the app's job);
Microsoft OAuth; aggregations; multi-agent orchestration.

## Tests
Each item ships with focused tests; provider tool-calls + OAuth exchanges are
injected/faked offline.
